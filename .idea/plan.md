# 2026 너드살롱 Vol.2 발표자료 계획

## 목표

`src/pages/projects/nerdsalon-2026-vol2/index.astro`에 2026년 두 번째 너드살롱 발표인 **민주주의 수학적 불가능성**을 위한 인터랙티브 발표자료 허브를 만든다.

이 프로젝트는 하나의 Marp 발표 HTML과 여러 개의 인터랙티브 자료로 구성한다. 메인 페이지는 발표자료와 각 인터랙티브 자료로 이동하는 링크만 제공한다. 전체 스타일은 Pretendard 폰트와 매우 기본적인 CSS만 사용하며, `https://justfuckingusecss.com/`에 가까운 단순한 방향을 따른다.

화려한 웹앱보다 발표 현장에서 바로 이해하고 조작할 수 있는 “너드스러운 실험실형 자료”가 목표다.

## 발표 구성

### 0. 문제 인식

- 2000년 미국 대통령 선거를 출발점으로 삼는다.
- 우리 모두 이미 다수대표제 투표의 문제점을 어느 정도 알고 있다는 인식에서 시작한다.
- 단순히 “나쁜 제도”라고 말하기보다, 왜 제도 설계 문제가 수학적 문제로 이어지는지 소개한다.

### 1. 여러 투표제

결선투표제, 드보르다 투표제, 콩도르세 투표제를 차례로 설명한다. 각 제도는 다수대표제보다 나아 보이지만, 모두 고유한 실패 사례가 있음을 보여준다.

- 결선투표제: 어떤 후보에게 지지율이 낮아질 만한 사건이 생겼는데도 오히려 당선 확률이 올라가는 역설을 보여준다.
- 드보르다 투표제: 유권자의 실질 의사와 직접 관련 없는 후보의 존재나 순위 배치가 결과에 영향을 주는 문제를 보여준다.
- 콩도르세 투표제: 모든 후보를 1:1로 비교해도 승리자를 절대 낼 수 없는 콩도르세 루프를 보여준다.

### 2. 투표제의 수학적 모델

- 길보의 모형과 애로의 사회후생함수를 중심으로 투표제를 수학적으로 정의한다.
- 유권자 개인의 선호 순서가 사회 전체의 선호 순서로 변환되는 과정을 모델링한다.
- 이후 애로의 불가능성 정리를 이해하기 위한 언어와 조건을 정리한다.

### 3. 애로의 불가능성 정리

- 합리적으로 보이는 조건들을 동시에 만족하는 완벽한 사회후생함수는 존재하지 않는다는 결론을 설명한다.
- 단순한 철학적 주장이나 정치적 냉소가 아니라, 엄밀한 수학적 정리라는 점을 강조한다.

### 4. 던컨 블랙의 정리

- 단봉 선호와 중위 투표자 정리를 소개한다.
- 특정한 조건에서는 중도층 투표자의 의견이 사회의 최선을 반영할 수 있음을 설명한다.
- 애로의 불가능성 정리 이후에도 민주주의가 항상 무의미해지는 것은 아니라는 균형점을 제공한다.

### 5. 대안적 투표제와 갈등 완화

- Approval voting system과 rated voting system을 소개한다.
- 관련 연구자료를 바탕으로 이런 투표제가 사회적 갈등을 줄일 수 있는 가능성을 설명한다.
- “완벽한 투표제”가 아니라 “더 나은 실패 방식”을 찾는 문제로 정리한다.

### 6. 결론

- “민주주의는 다른 모든 것들보다 나은 최악의 시스템이다.”라는 메시지로 마무리한다.
- 수학적 불가능성은 민주주의 포기의 근거가 아니라, 제도 설계를 더 겸손하고 정교하게 해야 한다는 근거로 다룬다.

## 인터랙티브 자료 계획

발표 HTML을 먼저 완성한 뒤 인터랙티브 자료의 정확한 개수와 세부 구현을 확정한다. 다만 1번, 2번, 3번, 4번 주제에 대해서는 반드시 인터랙티브 자료를 만든다.

### 투표제 체험

발표 장소에서 직접 여러 투표제를 체험할 수 있게 한다. 최소한 아래 세 가지의 차이는 같은 예시 데이터로 비교할 수 있어야 한다.

- 다수대표제
- 결선투표제
- Approval voting, 승인투표제

예시는 “야식으로 먹고 싶은 음식”을 사용한다. 발표 참여자가 후보 음식에 투표하거나 선호를 입력하면, 투표제별로 결과가 어떻게 달라지는지 보여준다.

### 여러 투표제의 실패 사례

- 결선투표제 자료: 특정 후보의 지지율 하락이 오히려 당선 가능성을 높이는 상황을 조작해 볼 수 있게 한다.
- 드보르다 자료: 유권자의 핵심 선호가 그대로여도 무관한 후보의 추가나 순위 조정으로 결과가 바뀌는 상황을 보여준다.
- 콩도르세 자료: A가 B를 이기고, B가 C를 이기고, C가 A를 이기는 순환 구조를 직접 확인할 수 있게 한다.

### 수학적 모델과 정리

- 수학적 모델링 자료: 유권자별 선호 순서를 입력하면 사회적 선호 또는 승자가 어떻게 계산되는지 보여준다.
- 애로의 불가능성 정리 자료: 몇 가지 공정성 조건을 켜고 끄며 동시에 만족이 어려운 이유를 관찰하게 한다.
- 던컨 블랙 정리 자료: 단봉 선호 분포에서 중위 투표자의 위치가 결과를 어떻게 결정하는지 시각화한다.

## 구현 방향

- 대상 페이지는 `src/pages/projects/nerdsalon-2026-vol2/index.astro`의 독립 Astro 라우트로 유지한다.
- 블로그 공통 레이아웃, Tailwind 컴포넌트, 마케팅 페이지식 카드 UI는 사용하지 않는다.
- 기존 `src/static/fonts/PretendardVariable.woff2`를 사용해 Pretendard를 적용한다.
- CSS는 페이지 내부에 최소한으로 작성한다.
- 메인 페이지는 제목, 짧은 설명, 발표자료 링크, 인터랙티브 자료 링크 목록만 제공한다.
- Marp 발표자료는 `src/pages/projects/nerdsalon-2026-vol2/slide.md`를 원본으로 작성하고, 옆의 `slide.html`로 실시간 변환되는 정적 산출물을 메인 페이지에서 링크한다.
- Astro 페이지 내부의 클라이언트 스크립트는 필요한 인터랙션에만 제한적으로 사용한다.

## 확인 기준

- `/projects/nerdsalon-2026-vol2`에서 메인 페이지에 접근할 수 있어야 한다.
- 메인 페이지의 모든 링크는 실제 발표자료 또는 인터랙티브 자료로 연결되어야 한다.
- `pnpm build`가 성공해야 한다.
- 모바일과 발표용 화면에서 텍스트가 겹치지 않아야 한다.
- Pretendard가 적용되어야 한다.
- 투표제 체험 자료는 최소한 다수대표제, 결선투표제, 승인투표제의 결과 차이를 비교할 수 있어야 한다.

이번 단계에서는 전체 기획 이해를 문서로 정리한다. 발표자료 본문 작성, Marp 변환 파이프라인 구성, 인터랙티브 자료 구현은 이후 단계에서 진행한다.
